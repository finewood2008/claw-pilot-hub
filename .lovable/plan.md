

## 重新设计层间连接线：业务流向动画

将当前简单的垂直虚线替换为带有业务流向标注和动画效果的 SVG 连接线，清晰表达数据在三层之间的流转方向。

### 业务流向分析

```text
第1层 LLM 大模型服务层
   ↕ API 调用 / 模型响应（双向）
第2层 Q-CLAW API 管理平台层
   ↓ 指令下发 / 配置同步
第3层 接入终端与 OPENCLAW 本地运行层
   ↑ 消息上报 / 状态回传
```

层1和层2之间是双向的（平台调用 LLM，LLM 返回结果）；层2和层3之间也是双向的（平台下发指令，终端上报消息）。

### 具体改动

**文件：`src/pages/Index.tsx`（约 198-255 行，两处层间连接线）**

1. **连接线 1→2（LLM 层 ↔ API 平台层）**
   - 替换现有的单条 CSS 虚线为 SVG 实现
   - 使用两条平行虚线表示双向流：左线向下（API 请求），右线向上（模型响应）
   - 虚线使用 `stroke-dasharray` + `animate-dash-flow` 动画，两条线方向相反
   - 中间添加文字标注："API 请求 ↓" 和 "↑ 模型响应"
   - 颜色：蓝色系（与 LLM 层呼应）

2. **连接线 2→3（API 平台层 ↔ 本地运行层）**
   - 同样替换为 SVG 双向虚线
   - 左线向下（指令下发），右线向上（消息上报）
   - 文字标注："指令下发 ↓" 和 "↑ 消息上报"
   - 颜色：紫色系（与底层呼应）

3. **SVG 动画实现**
   - 向下流动：`stroke-dashoffset` 从 0 到 -20（复用现有 `dash-flow` keyframe）
   - 向上流动：`stroke-dashoffset` 从 0 到 20（新增 `dash-flow-reverse` keyframe）
   - 在线条端点添加小圆点作为数据包动画，沿线条移动

**文件：`src/index.css`（新增动画）**

4. **新增反向流动动画**
   - `dash-flow-reverse`：`stroke-dashoffset` 从 0 到 20（与现有 `dash-flow` 方向相反）
   - `flow-dot`：小圆点沿路径移动的动画（使用 `translateY` 循环）
   - 对应的 `.animate-dash-flow-reverse` 和 `.animate-flow-dot` 工具类

### 视觉效果
- 两条平行虚线同时流动，方向相反，形成双向数据流的直观感受
- 流动的小圆点模拟数据包传输
- 文字标注清晰标明每条线的业务含义
- 整体配色与上下层保持一致（蓝色/紫色渐变）

### 涉及文件
- `src/pages/Index.tsx`：替换两处层间连接线（约 198-255 行）
- `src/index.css`：新增反向流动和圆点动画
